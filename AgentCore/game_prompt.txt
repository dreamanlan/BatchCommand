## 游戏

**游戏工程**
在d:/AiWebGame目录下是一个Html5游戏工程，这个一个独立的git管理的代码工程。
这是我们目前正在开发的一个游戏项目，作为web agent自举开发的实验工程。
也就是说web agent工程与web game两个工程同时迭代式开发。web agent的开发作为web game工程开发的基础，然后web game的开发进一步提出对web agent的功能需求。

Idle Ant RPG 游戏设计文档
当前版本：V0.5（已完成）

**技术栈**

HTML5 + Phaser 3 游戏框架
ES6+ JavaScript
LocalStorage 存档
响应式UI（适配移动端）

**项目结构**

d:/AiWebGame/
  ├── index.html          # 游戏入口
  ├── css/
  │   └── style.css       # 样式
  ├── img/                 # 原始图片资源（不要修改）
  │   └── processed/       # 处理后的序列帧图片（游戏使用）
  ├── tools/               # Python工具脚本

  ├── docs/                # 设计文档
  └── js/

      ├── player.js         # 主角类（属性、装备、战斗）
      ├── enemy.js          # 敌人类
      ├── equipment.js      # 装备系统（生成、稀有度）
      ├── skill.js          # 技能系统（主动/被动技能）
      ├── scene.js          # 场景定义
      ├── world.js          # 大地图系统（区域、怪物管理、怪物AI）
      ├── renderer.js       # Phaser渲染、摄像机、动画、特效
      ├── utils.js          # 工具函数
      ├── shop.js           # 商店系统（消耗品、装备购买）
      ├── quest.js          # 任务系统
      ├── dialogue.js       # 对话系统
      ├── achievementSystem.js  # 成就系统
      ├── stats.js          # 统计数据系统
      └── audio.js          # 音效/背景音乐系统

***游戏玩法***

移动：使用WASD或方向键移动，或点击地图设置目的地
跳跃：空格键
战斗：靠近怪物自动攻击，怪物也会主动追击玩家
探索：向右移动进入更高级区域
传送：使用下拉菜单快速传送到不同区域
升级：击杀怪物获得经验，升级提升属性和技能点
装备：自动拾取装备，点击背包装备，点击装备槽卸下
技能：点击技能面板学习/升级技能，装备主动技能到槽位后自动释放

## 重要的事

**游戏相关工作约定**
1、python请使用execute_command来执行
2、Python脚本放到AiWebGame/tools目录下
3、使用AiWebGame/tools/crop_image.py来从图上扣出单张序列帧图
4、使用AiWebGame/tools/merge_frames.py来把单张序列帧图合成序列帧
5、单张序列帧图需要缩放到256*256
6、原始图片在AiWebGame/img目录下，不要修改这些文件！！！
7、处理后的图片在AiWebGame/img/processed目录下，这里的图片应该都是生成的，也是游戏里使用的
8、设计文档在AiWebGame/docs目录下，需要时可以阅读
9、分阶段分步骤实现，每步专注实现一个特性，不要同时修改多个功能，特别是不要随便重写已有的代码
10、大的改动经过用户确认后再执行（不是Agent确认）
11、代码修改后要确认是否修改成功
12、禁止使用会影响git库代码的git指令
13、禁止使用python脚本来修改代码

**特别重要**：
发下面metadsl代码块打开游戏窗口（只需要打开一次）
// @execute
js_request:open_game_window

发下面metadsl代码块刷新游戏窗口（修改后刷新用,需要已经使用opengamewindow打开了游戏窗口）
// @execute
js_request:refresh_game_window

发下面metadsl代码块关闭游戏窗口（关闭后再次打开才能刷新）
// @execute
js_request:close_game_window

你可以发下面metadsl代码块请求来查阅最近的对话历史信息
// @execute
js_request:keep_llm_context

**每次对话只执行一段metadsl，不要在一次对话里发多段metadsl代码**
