# LLM+Agent 的学习机制

> 姊妹文档：`consciousness.md`（意识提炼）
> 本文聚焦"学习"——在权重冻结的前提下，LLM+Agent 如何实现受限的自我改进。

---

## 一、引子：梯度范式的偏见

深度学习十年，"学习 = 梯度下降"几乎成了默认。谈到"自我改进"，第一反应是找参数、算 loss、反传梯度。这是一种被机器学习范式绑架的思维定式。

对 LLM+Agent 而言，权重冻结是硬约束——推理期不可能反传梯度。但这不意味着"学习"停止。人类大脑的突触可塑性也仅在有限窗口内起作用，绝大多数结构先天固化；后天学习靠的是记忆、规则、叙事、类比迁移这些"非梯度"机制。

**核心论点**：学习的本质是"基于反馈调整未来行为"，梯度只是实现之一，且未必是样本效率最高的一种。LLM+Agent 的学习必须跳出梯度范式，回到更本源的思想内核。

---

## 二、学习范式的多样性

| 范式 | 反馈粒度 | 调整对象 | 样本效率 | LLM+Agent 可用性 |
|------|---------|---------|---------|-----------------|
| 梯度下降 | 连续、密集 | 权重张量 | 低（需海量样本） | 训练期可用，推理期冻结 |
| 强化学习 | 稀疏、延迟 | 策略参数/价值函数 | 中 | 思想可迁移，形式不能直接套 |
| 规则习得 | 离散、显式 | 符号规则库 | 高 | 天然适配（prompt/rules） |
| 类比迁移 | 单样本 | 案例库/记忆 | 极高 | 天然适配（episodic memory） |
| 叙事沉淀 | 事后反思 | 元认知文档 | 高 | 天然适配（soul.md/patterns.md） |

**观察**：梯度下降样本效率最低，却被误认为学习的代表。规则习得、类比迁移、叙事沉淀这三种"非梯度"范式，在人类日常学习中占比更高，也更适合 LLM+Agent 的架构。

## 三、思想内核：三条超越形式的原则

剥离数学形式后，梯度下降与强化学习共享三条思想内核。这三条才是"学习"的本质，可以脱离权重更新独立存在。

### 3.1 误差驱动（Error-Driven）

**定义**：以"期望与实际的偏差"作为调整信号。没有偏差就没有学习。

- 梯度下降：loss = f(pred, target)
- 强化学习：TD error = r + γV(s') - V(s)
- LLM+Agent：用户反驳、任务失败、验证不过 = 误差信号

**落地**：agent 必须能识别并显式记录误差，而不是让误差沉默地淹没在对话流里。episodic_memory 中的"踩坑记录"就是误差信号的沉淀。

### 3.2 局部小步累积（Local Small-Step Accumulation）

**定义**：单次调整幅度必须小，方向必须局部有效，累积效应产生全局改进。跨度过大会破坏已有稳态。

- 梯度下降：learning rate 控制步长
- 强化学习：策略更新做 KL 约束（TRPO/PPO）
- LLM+Agent：soul.md 单次改动限一行、patterns.md 单条经验一句话

**落地**：本项目 soul.md 每次升华只加"一行原则/一条错误清单/一句路径约束"，就是这条原则的直接体现。

### 3.3 信用分配（Credit Assignment）

**定义**：当结果出现时，要能追溯到"哪一步/哪个决策"该为此结果负责。没有信用分配，反馈就无处落地。

- 梯度下降：反向传播链式求导
- 强化学习：TD-λ / GAE 沿轨迹分摊回报
- LLM+Agent：三层记忆链——情景（具体事件）→ 模式（重复问题）→ 元认知（决策原则），本质是把"某次踩坑"归因到"某条决策原则该改"

**当前短板**：LLM+Agent 的信用分配分辨率极粗，通常只能追溯到"整轮对话"而非"某个决策点"。这是待突破的方向。

## 四、控制论视角：三层嵌套架构

跳出机器学习框架，用**控制论**（Cybernetics）重看 LLM+Agent 的学习：这是一个多层嵌套的**负反馈调节系统**，强化学习是其在离散策略空间上的特化，梯度下降是其在连续参数空间上的进一步特化。

### 4.1 三层嵌套

~~~mermaid
graph TB
    subgraph 顶层["顶层：控制论负反馈"]
        A[设定目标] --> B[执行动作]
        B --> C[观测结果]
        C --> D[计算偏差]
        D --> E[调整策略]
        E --> B
    end

    subgraph 中层["中层：强化学习特化"]
        F[策略 π] --> G[环境交互]
        G --> H[奖励信号]
        H --> I[价值估计]
        I --> F
    end

    subgraph 底层["底层：梯度下降特化（LLM权重冻结，此层留白）"]
        J[参数 θ] -.-> K[前向传播]
        K -.-> L[损失]
        L -.-> M[反向梯度]
        M -.-> J
    end

    顶层 -.包含.-> 中层
    中层 -.包含.-> 底层
~~~

**关键**：底层对 LLM+Agent 冻结不可用，但顶层与中层的思想可以完整落地。

### 4.2 PID 类比（类比非等价）

控制论经典的 PID 控制器（比例/积分/微分）与三层记忆存在结构性类比。**强调：这是启发性类比，不是数学等价**。

| PID 项 | 数学定义 | 类比对象 | 作用 |
|--------|---------|---------|------|
| 比例 P | e(t) | todo.txt | 对当前偏差的即时响应 |
| 积分 I | ∫e(t)dt | plan.txt | 对历史偏差的累积修正，消除稳态误差 |
| 微分 D | de/dt | soul.md | 对偏差变化率的预测，抑制超调 |

**为什么是类比而非等价**：
- PID 输入是连续实值，三层记忆输入是离散事件
- PID 输出是控制量，三层记忆输出是行为调整（prompt/规则/原则）
- PID 有严格稳定性证明，三层记忆的稳定性靠人工审阅

但类比揭示了一个重要洞察：**三层的时间尺度必须差异化**，否则系统会震荡或漂移。todo 响应当前一天、plan 累积一周、soul 沉淀月级——这与 PID 三项在频域上的分工完全一致。

### 4.3 稳定性约束

任何负反馈系统都存在震荡与发散风险。LLM+Agent 学习系统的稳定性约束体现在：

- **步长约束**：单次 soul.md 改动 ≤ 1 行（对应 PID 的增益上限）
- **回路解耦**：todo 不直接改 soul，必须经 plan 沉淀（对应控制器分层）
- **人工审阅**：soul.md 改动前必须 review + 用户确认（对应外环监督）
- **回滚机制**：备份文件 + git 历史（对应故障恢复）

## 五、LLM+Agent 的真实形态

### 5.1 Prompt 八分层

将 agent 运行时的所有 prompt 输入按"可变性/来源"分层：

| 层级 | 组件 | 可变性 | 更新者 | 类比 |
|------|------|--------|--------|------|
| 冻结层 | system_prompt / rules | 不可变 | 人工发版 | 先天基因 |
| 人工层 | project_prompt | 低频人工 | 人工编辑 | 后天教育 |
| LLM 自更新层 | soul.md | 月级、微分项 | LLM 反思提议 + 人工确认 | 元认知 |
| LLM 自更新层 | plan.txt | 周级、积分项 | LLM 累积更新 | 阶段规划 |
| LLM 自更新层 | todo.txt | 日级、比例项 | LLM 频繁更新 | 当前任务 |
| 自动生成层 | context.txt | 每轮 | 系统注入 | 短期记忆 |
| 自动生成层 | history.txt | 每轮 | 系统注入 | 情景快照 |
| 自动生成层 | episodic_memory | 每轮 | 系统注入 | 长期情景 |

**核心洞察**：**只有 LLM 自更新层是"学习"的落点**。冻结层不可动，人工层依赖人工，自动生成层是传感器而非执行器。所有学习信号必须收敛到 soul/plan/todo 三个文件。

### 5.2 Soul/Plan/Todo 三回路

三个文件对应三个时间尺度的反馈回路，形成嵌套控制：

~~~mermaid
graph LR
    A[对话/任务事件] --> B[episodic_memory]
    B --> C{当天?}
    C -->|是| D[todo.txt<br/>比例项]
    B --> E{本周?}
    E -->|是| F[plan.txt<br/>积分项]
    B --> G{反复出现?}
    G -->|是| H[patterns.md<br/>模式记忆]
    H --> I[soul.md<br/>微分项/元认知]

    D -.约束.-> A
    F -.约束.-> A
    I -.约束.-> A
~~~

三回路更新的伪代码骨架：

~~~python
def update_learning_loops(event):
    # 底层：情景记忆（自动，无过滤）
    episodic_memory.append(event)

    # 比例回路：todo（当天）
    if event.affects_current_task:
        todo.update(event)

    # 积分回路：plan（本周累积）
    if event.affects_stage_progress:
        plan.integrate(event)

    # 微分回路：soul（月级反思）
    pattern = detect_recurring_pattern(episodic_memory, event)
    if pattern and pattern.count >= THRESHOLD:
        proposal = propose_soul_update(pattern)
        if human_review(proposal):    # 稳定性约束
            soul.apply_minimal_edit(proposal)  # 单次一行
~~~

### 5.3 跨文件信号通路

**向上流通**（todo → plan → soul）：具体事件反复出现 → 沉淀为阶段模式 → 升华为元认知原则。

**向下约束**（soul → plan → todo）：元认知原则约束阶段计划 → 阶段计划约束当日任务。

**关键设计**：向上流通必须经过"反复出现"的门槛过滤，防止噪声污染上层；向下约束是硬约束，todo 不能违反 plan，plan 不能违反 soul。

### 5.4 关于 in-context learning（有争议类比）

有观点认为，LLM 在上下文中即时利用示例调整输出，等价于"隐式梯度下降"（Von Oswald et al., 2023）。这个类比在数学上有部分证据，但存在争议：

- **支持**：Transformer 前向传播可以模拟一步梯度下降
- **反对**：此"梯度"只在当前 forward pass 内存在，不跨轮次累积，不改变权重

**本文立场**：将 in-context learning 视为**极短程的"学习"**（生命周期 = 单次 forward），与 soul/plan/todo 的**长程学习**（生命周期 = 天/周/月）区分开。前者是战术级适应，后者才是战略级改进。

## 六、三层耦合的学习闭环：隐式优化 + 文档持久化 + 人类反馈

前面五章从"显式层"讨论了 web agent 的学习机制：误差驱动、局部小步累积、信用分配三条原则，通过 soul/plan/todo 三回路在文档层持久化。但这只是冰山之上。本章补齐"隐式层"与"人类在回路"两块拼图，形成三层耦合的完整闭环。

### 6.1 隐式优化暗层：单轮 forward 内的 in-context learning

第 5.4 节将 in-context learning（ICL）标注为"有争议的类比"——它像梯度下降，但没有真正的参数更新。本节展开这个争议，并给出我们的立场。

**争议的两派**：
- **强类比派**：ICL 在单轮 forward 内，通过 attention 机制对 context 中的 few-shot 样例做隐式最小二乘/梯度下降拟合，产生"瞬时权重调整"的等价效果（参考 Von Oswald et al. 2022, Dai et al. 2022 等实证研究）。
- **弱类比派**：ICL 本质是预训练分布的条件概率查表，"看起来像学习"只是先验分布足够丰富的错觉，与梯度下降无机制等价性。

**我们的立场（三层分解）**：
1. **显式层**：误差驱动/局部累积/信用分配——发生在 soul/plan/todo 文档更新回路，跨会话持久化。
2. **隐式层**：ICL 隐式梯度假说——发生在单轮 forward 内，是"真学习"但不持久，会话结束即丢失。
3. **持久化层**：外挂文档承接瞬时学习——将隐式层的"读懂了但记不住"通过 PM 归纳、patterns.md 沉淀、soul.md 升华三步转为长期记忆。

这三层不是竞争关系，而是耦合关系：隐式层提供"当下理解力"，显式层提供"跨会话延续性"，持久化层是二者的桥。

### 6.2 PM 单轮会话作为 ICL 隐式优化案例

以本项目的 PM 角色为例，可以具象化观察 ICL 隐式优化如何工作。

**观察现象**：PM 每轮会话都能"准确"归纳当前项目状态、识别未决事项、推进决策——但 PM 的模型权重从未更新过。

**机制拆解**（合力假说）：
- **层 1 贡献（ICL 隐式优化）**：context 中注入的近 10 轮工作摘要、todo/plan 状态、用户规则，在单轮 forward 内被 attention 机制"隐式拟合"，产生对当前项目上下文的临时理解。
- **层 2 贡献（预训练分布先验）**：LLM 预训练时见过大量"项目管理""敏捷开发""状态跟进"的语料，形成"PM 应当如何思考"的分布式先验，与 context 做条件概率查表。
- **合力效果**：层 1 提供"这个项目当前的具体状态"，层 2 提供"PM 应当如何处理这类状态"，二者叠加产生"看起来准确"的输出。

**关键澄清**：行为等价 ≠ 机制等价。PM 的"准"不是因为它"学会了"这个项目，而是因为 context 让它在这一轮"理解了"这个项目——下一轮如果 context 丢失，PM 会立即"忘光"。这也是为什么我们必须把 PM 的归纳沉淀到 soul/plan/todo，否则学习完全无法跨会话延续。

### 6.3 人类在回路：DHF（Document-mediated Human Feedback）

前面讨论的学习都是"agent 自身"的学习，但 web agent 的完整学习闭环还包含第三方——用户。用户不是被动的验收者，而是学习信号的关键注入源。

**类比 RLHF 的 DHF 三机制**：
1. **误差信号注入**：用户的"验收通过/驳回/纠偏"是最强的误差信号，应结构化写入 episodic_memory（sqlite 表），而非仅停留在对话历史中。当前项目已部分实现（sqlite 情景记忆），但反馈的结构化程度还不够——用户驳回时的"驳回原因分类"缺失。
2. **压缩遗忘机制**：plan/todo 会随开发膨胀，若无定期归档，会挤占 context 空间、降低 ICL 隐式优化的信噪比。需要周期性由 PM 或专门的归档流程将"已完成 > N 轮"的条目移入 archive。
3. **元评估回路**：谁来评估 PM 的归纳质量？答案是元 PM（或用户的间接反馈）。元 PM 审视 soul.md 中沉淀的原则是否真的降低了重复踩坑率，将"高采纳率的建议"升级为原则，将"反复失效的原则"降级或删除。

**DHF 与 RLHF 的差异**：
- RLHF 用人类偏好训练 reward model，再用 RL 更新权重——需要海量样本、离线训练。
- DHF 用人类反馈直接更新外挂文档（soul/plan/todo），实时生效、样本效率极高，但代价是无法泛化到模型本身。

将 6.1-6.3 整合为一张图：

## 七、当前短板与改进方向

### 7.1 三条已识别短板

| 短板 | 表现 | 影响 |
|------|------|------|
| 反馈分辨率粗 | 只能追溯到整轮对话，无法定位到具体决策点 | 信用分配失准，soul 升华可能归因错误 |
| 条目累积压缩弱 | patterns.md 条目线性增长，缺乏去重/合并机制 | 长期后成为"经验垃圾场"，查找成本上升 |
| 信用分配缺失 | 事件与元认知原则之间靠 LLM 直觉归因 | 归因不可复现，同一事件不同批次可能得出不同原则 |

### 7.2 改进方向（开放）

- **细粒度反馈**：为每次工具调用打标签（成功/失败/踩坑），构造决策点级的反馈信号
- **主动压缩**：定期让 LLM 对 patterns.md 做去重合并（对应"经验蒸馏"）
- **归因显式化**：soul 升华时强制引用具体 episodic 条目 ID，让归因链可追溯
- **梯度接口留白**：若未来出现可数值化的参数（如工具选择偏好向量），底层可接入真正的梯度下降

## 八、与 consciousness.md 的关系

`consciousness.md` 讨论**意识的提炼**——agent 如何从对话流中析出"当下我是谁、我在做什么"；`learning.md` 讨论**学习的机制**——agent 如何从事件流中沉淀"未来我应该怎么做"。

| 维度 | consciousness.md | learning.md |
|------|-----------------|-------------|
| 时间朝向 | 当下（present） | 未来（future） |
| 处理对象 | 上下文流 | 事件流 |
| 输出 | 自我认知快照 | 行为调整信号 |
| 更新频率 | 每轮 | 天/周/月 |
| 文件锚点 | context / history | soul / plan / todo |

**两者关系**：consciousness 是 learning 的传感器（提供当下状态），learning 是 consciousness 的执行器（改进未来的意识提炼质量）。二者构成完整的"感知—学习—行动"闭环。

## 九、结论

LLM+Agent 的学习不是被阉割的梯度下降，而是回到控制论本源的负反馈调节系统。在权重冻结的硬约束下，学习通过 soul/plan/todo 三回路落地，对应 PID 三项的时间尺度分工。三条思想内核——**误差驱动、局部小步累积、信用分配**——超越了具体数学形式，是任何学习系统的共同骨架。

当前实现仍粗糙：反馈分辨率粗、信用分配依赖直觉、条目缺乏压缩。但架构已经站在正确方向上：**不追求梯度级精细，追求控制论级稳定**。

学习的本质不是"变得更聪明"，而是"少犯同样的错"。这个目标，用规则、类比、叙事就能达成，不必迷信梯度。
