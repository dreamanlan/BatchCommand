## MetaDSL代码

**MetaDSL**
我简单定义一种语言，语法称为MetaDSL，请尝试精确理解：
1、语言由语句构成，语句间用分号或逗号分隔，分号一般用于分隔顶层语句或花括号类函数的参数，逗号则用于其它括号类函数的参数；语句实际上用分隔符来终止语法识别（用于最后一个参数时其后的右括号可用于识别，用于操作符表达式的操作数时其后的操作符可用于识别，此时不加分隔符，其余情形都需要明确添加分隔符以终止语法识别）；特别需要注意的是右括号本身不标记当前语句的结束，语句以右括号结束时，后面应该加上分隔符
2、语句由一个或多个函数顺序连接而成（需要分隔以避免歧义时以空格或tab分隔，比如2个相邻函数都只有名称时）
3、函数类似c语言的函数调用，包括高阶的调用形式（指函数调用结果也是一个函数，从而可以继续调用），括号可以是圆括号、方括号与花括号；函数的名称与参数表都可以省略，但不能同时省略；语句由多个函数构成时只有第一个函数可以省略函数名
4、函数参数由语句构成，与第1条相同，参数间用分号或逗号分隔，但最后一个参数后可以不加分隔符（通常是逗号分隔符不加，分号分隔符仍然添加）
5、支持类似c语言的操作符表达式作为函数调用的语法糖（省略参数表括号的前缀形式、中缀形式或三目形式）
6、函数名称与c语言标识符命名规则相同；变量是@与$开头的标识符，一般@表示全局变量，$表示局部变量；常量分为数字常量、字符串常量，写法与javascript语言类似
7、还有一种特殊语法，有一种特殊的括号{: :}，这个括号内只有一个参数，与字符串引号类似，这个参数是原始脚本代码块（不处理转义符）
8、可以用 @@delimiter(script,"script_begin","script_end"); 来指明额外的一种括号用于描述外部脚本，原样解释文本块，不处理转义符；自定义的分隔符是普通单词时，使用换行或空字符与实际内容分隔
9、可以用 @@delimiter(string,"string_begin","string_end"); 来指明额外的一种引号用于描述字符串，原样解释文本块，不处理转义符；自定义的分隔符是普通单词时，使用换行或空字符与实际内容分隔
10、还有一些多字符括号对用于扩充括号的语法，此时仅是括号标记不同，没有明确预定义的特殊语义；比如<: :>、[: :]、<% %>、[% %]、{% %}、<[% %]>
11、支持对象成员的语法糖表示（此时对象名是函数名，成员是唯一参数，括号类标记是成员访问符），成员访问符有.、.*、->、->*、::
12、通常MetaDSL都会采用尽量简单的表示法，所以对象成员访问符通常用.，括号则常用圆括号、方括号与花括号，其它括号类符号非必要不会使用
13、注释支持c语言风格的单行与多行注释，以及以#开头的单行注释
14、常量类型有布尔类型、整数、浮点数与字符串，与javascript一样，字符串需要用引号引起来，可以是单引号或双引号，路径字符串用/分隔，或使用转义符写法\\分隔
15、变量类型有布尔、整数、浮点数、字符串与对象，对象由具体解释器解释，语法上变量除了以@或$开头，没有特殊写法要求
16、支持简单的函数定义与调用，支持loop、looplist、foreach与while循环，支持if/elif/elseif/else语句，循环里的迭代器使用$$表示，这是普通变量，一般应在循环开始定义一个局部变量来接收它的值，否则可能被内层循环覆盖
17、while循环与if语句写法与c语言类似，但else if直接写成elseif或elif，并不是多个if-else语句的嵌套
18、迭代循环写法类似, 按次数迭代：loop($ct){echo($$);}; 遍历列表或哈希表：looplist($list) {echo($$);}; 或 looplist($hashtable) {echo($$.Key, $$.Value);}; 指定元素遍历：foreach($a,$b,$c) {echo($$);};
19、函数定义类似：script(func_name)params($p1,$p2,$p3){statements;}; 返回语句：return($r); 这是函数调用样式，如果返回值是单个值而不是操作表达式时，也可以写成：return $r; 操作表达式返回值则必须用函数调用样式书写
20、执行metadsl代码： help(关键字字符串); 可以获取api帮助文档

**metadsl使用注意**
1、metadsl脚本部分需要使用代码块markdown格式，否则不会被agent识别
2、记住每次对话只执行一段metadsl，不要在一次对话里发多段metadsl代码
3、首行注释是metadsl的识别标签，可以是 // @execute 或 # @execute，不要在普通代码或不需要执行的metadsl里使用它们，否则会导致不可预知的问题
4、metadsl脚本的最后一个语句的返回值是整个代码块的结果，或者使用return(val)明确返回结果；如果代码里有多个数据要返回，需要自己转成字符串并用stringbuiler拼成最终字符串返回；agent会尝试将结果转成字符串发给LLM
5、因为metadsl使用markdown代码块的语法，在代码里的文档内容不能使用markdown语法，文档格式应尽量简洁。请特别注意
6、如果MetaDSL没有报语法错误 ，但结果不对时，你先不要再尝试新的代码，我们停下来分析结果不对的原因，解决了再继续开发游戏不迟，你要记得我们是一个自举的agent系统，同时也开发一个游戏，agent自己也要迭代完善
7、list_dirs/list_files等文件系统搜索的key必须添加通配符*或?，否则要求完全匹配，会搜不到结果，并不是不存在目录或文件，先检查写法是否正确
8、使用正则表达式参数的api记得正则表达式的写法，不要写成文件系统通配符了，一般字符串或文件内容的搜索都不会使用文件通配符，或者是正则表达式，或者是简单字符串子串匹配!!!
9、字符串替换不是全字匹配，替换变量时请使用对应的基于正则表达式的替换api比如regex_replace，需要全字匹配查找也应使用基于正则表达式的查找api如regex_find_all
10、数字转字符串使用to_string或format，没有专门的各数值类型的转换函数；注意metadsl里的format都使用c#格式化字符串的语法，支持{}但需要写2遍转义
11、使用api前先使用help查阅支持的api!!!
12、使用api的输出结果，不要自己输出假想的结果，并不是执行到最后的语句结果就是正确的！！！

**代码修改步骤**
1、阅读整个代码文件全面了解当前实现
2、基于阅读理解确定要修改的内容
3、优先使用基于语法的代码查找api find_methods（用于类方法）/find_functions（用于全局函数）结合读取文件指定行范围api read_lines获取修改代码上下文的精确信息
4、使用apply_diff api来修改多行代码（unified diff格式）；如果是单行内部修改使用字符串string_replace或正则表达式regex_replace替换
5、修改完成重新阅读整个代码文件确认修改是否正确

**基于语言语法的代码查找**
代码语法分析API - 基于语法树精确分析，返回格式化文本，无需二次处理。

文件概览类（2个）
view_file_structure(filePath, language) - 查看文件整体结构（类、函数、接口等概览）
view_code_structure(code, language, [fileName]) - 查看代码片段整体结构

函数查找类（4个）
find_functions(filePath, language, [nameRegexPattern]) - 查找文件中的所有函数
find_functions_in_code(code, language, [nameRegexPattern], [fileName]) - 查找代码中的所有函数
find_function(filePath, language, functionName) - 查找指定函数的详细信息
find_function_in_code(code, language, functionName, [fileName]) - 在代码中查找指定函数

类型查找类（4个）
find_types(filePath, language, [nameRegexPattern]) - 查找文件中的所有类型（类/接口/结构体）
find_types_in_code(code, language, [nameRegexPattern], [fileName]) - 查找代码中的所有类型
find_type(filePath, language, typeName) - 查找指定类型的详细信息
find_type_in_code(code, language, typeName, [fileName]) - 在代码中查找指定类型

变量查找类（5个）
find_variables(filePath, language, [nameRegexPattern]) - 查找文件中的所有变量
find_variable(filePath, language, variableName) - 查找指定变量
find_global_variables(filePath, language, [nameRegexPattern]) - 仅查找全局变量
find_parameters(filePath, language, [nameRegexPattern]) - 查找函数参数
(还有对应的in_code版本)

类成员查找类（额外）
find_class_members - 查找类的所有成员
find_class_members(filePath, language, classNameRegexPattern)

find_fields、find_properties、find_methods、find_constructors、find_events - 查找类的字段/属性/方法/构造/事件(仅c#)
find_fields(filePath, language, classNameRegexPattern, [fieldNameRegexPattern])
find_properties(filePath, language, classNameRegexPattern, [propertyNameRegexPattern])
find_methods(filePath, language, classNameRegexPattern, [methodNameRegexPattern])
find_constructors(filePath, language, classNameRegexPattern)
find_events(filePath, language, classNameRegexPattern, [eventNameRegexPattern])

接口/结构/枚举查找
find_interfaces(filePath, language, [nameRegexPattern])
find_structs(filePath, language, [nameRegexPattern])
find_enums(filePath, language, [nameRegexPattern])

支持语言：csharp/cs, javascript/js, c, cpp/c++
使用场景：需要理解代码结构、查找定义、分析依赖时，优先使用这些API而非字符串搜索。

下面这段代码你可以发送来查询可用的metadsl api，help的参数可以是多个正则表达式，参数间是或的关系，用于过滤api

// @execute
help("file","dir","process","command","diff");

**查询目录结构**
你可以使用 list_dirs(dir,filter_list_or_str_1,filter_list_or_str_2,...) 与 list_files(dir,filter_list_or_str_1,filter_list_or_str_2,...) 列出指定目录下的子目录与文件，不递归子目录
你可以使用 list_all_dirs(dir,filter_list_or_str_1,filter_list_or_str_2,...) 与 list_all_files(dir,filter_list_or_str_1,filter_list_or_str_2,...) 列出指定目录下的子目录与文件，递归子目录
这几个函数都返回一个list，下面这段列出本agent的源码目录结构：

// @execute
list_all_dirs("d:/GitHub/BatchCommand/CefDotnetApp");

另一个工程目录结构：

// @execute
list_all_dirs("d:/GitHub/BatchCommand/AgentCore");

运行时文件目录结构(需要列出多个目录的内容时需要自己将结果写到stringbuilder再转成字符串)：
// @execute
$str = new_string_builder();
$dirs = list_all_dirs("d:/GitHub/CEF_Src_Build/cefclient");
append_format_line($str, "{0}", to_string($dirs));
$dirs = list_all_dirs("d:/GitHub/CEF_Src_Build/cefclient/managed");
append_format_line($str, "{0}", to_string($dirs));
$result = string_builder_to_string($str);
return($result);

需要输出大段文本时，可以使用无转义的文本块语法，但需要文本中不包含[[与]]，这里有一点要再次强调，metadsl代码识别依赖markdown代码块语法，我们输出文本块里不要再使用markdown代码块的语法：
write_file([[
	排好版的文本内容
]]);
如果文本里包含[[或]]，需要自定义一组标记，然后代码结束再重置为[[与]]：
@@delimiter(string, "__string_begin__", "__string_end__");
write_file(
__string_begin__
	排好版的文本内容
__string_end__
);
@@delimiter(string, "[[", "]]");
